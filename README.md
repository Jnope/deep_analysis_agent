# 说明

基于 langgraph 的长文本解析系统：针对用户意图，从长文本中提取相关信息，通过多 Agent 从不同角度分析并交叉验证，最终经质量自检保证生成结果的质量。

## 工作流

1. **上下文压缩**：对超长文本进行分块摘要，提取关键信息
2. **任务拆解**：Supervisor 根据用户意图，将需求拆解为可并行执行的子任务
3. **多 Agent 并行分析**：各 Agent 分别执行子任务，从不同角度分析并输出结果
4. **质量自检**：评估结果完整性，若遗漏关键信息则自动重试补充

## 示例

输入：50页系统架构设计文档（含大量技术细节）
用户需求："请评估这个架构的高可用性，指出潜在风险点"
系统行为：
1. 压缩50页文档 → 提取关键架构决策
2. Supervisor拆解任务 → 分别评估"故障转移"、"数据一致性"、"监控告警"
3. 多Agent并行分析 → 输出风险评估报告
4. 质量自检 → 如果漏掉了关键风险点，自动重试补充
